## 2. Treesitter

- [ ] Decide whether to keep `nvim-treesitter`.
- Current recommendation: keep it for now.

### Why not remove it entirely?

- Neovim 0.12 includes core Treesitter integration and bundles only a small set of parsers.
- Current bundled parsers are limited compared to this config's language list.
- Core Nvim does not replace `nvim-treesitter`'s parser management workflow for the languages used here.
- This config currently installs parsers for `bash`, `fish`, `c`, `cpp`, `python`, `rust`, `go`, `json`, `yaml`, `toml`, and `glsl`, which goes beyond the bundled set.

### What can still be simplified?

- [x] Keep `nvim-treesitter`, but install/manage the plugin via `vim.pack`.
- [ ] Revisit whether `TSInstallFavorites` / `TSUpdateFavorites` / auto-install-on-enter are still worth keeping.
- [ ] Trim favorite parser list to languages actually used.
- [ ] Decide whether auto-install on first UI attach is desirable or if explicit install/update commands are cleaner.
- [ ] Re-check whether any functionality now covered by core Treesitter can replace custom mappings or glue code.

### When full removal becomes realistic

- [ ] Re-evaluate removal only if this config is happy with bundled parsers plus manual parser placement.
- [ ] Re-evaluate removal only if advanced `nvim-treesitter` query/module behavior is no longer needed.

## 4. Completion / Blink

- [ ] Decide whether Blink stays.
- [x] Pin Blink to the stable v1 release series instead of tracking arbitrary `main` commits.
- [x] Remove the stale Blink lockfile pin that referenced a non-tagged commit.
- [x] Force Blink prebuilt binary downloads to the stable `v1.*` release line for fresh installs.
- [ ] Run `vim.pack.update({ 'blink.cmp' })` on existing machines if the old Blink checkout is already installed on disk.

### Decision rule

- Keep Blink if:
  - richer completion UI matters;
  - multiple completion sources matter;
  - Rust fuzzy matcher quality/speed matters;
  - you want plugin-level completion behavior rather than built-in completion UX.

- Remove Blink if:
  - minimalism matters more than feature richness;
  - built-in LSP completion is now "good enough";
  - reducing plugin surface area is a priority.

### Current recommendation

- [~] Treat native 0.12 completion as viable now, not automatically better than Blink.
- [~] Blink is still likely better on UX and fuzzy behavior.
- [~] Native completion is now good enough to justify testing before deciding.

## Notes

- `vim.pack` is still marked experimental in Neovim docs, but the docs also say it should be stable enough for daily use.
- `nvim-treesitter` still has value on 0.12; core Treesitter is not the same thing as `nvim-treesitter`.
- The question is no longer "can Neovim do this at all?".
- The better question is "which parts are now built-in enough that custom glue is no longer worth carrying?".
